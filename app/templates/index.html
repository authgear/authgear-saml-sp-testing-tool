AuthnRequest Parameters

These parameters control how the SAML authentication request is sent to your Identity Provider.

When checked, the IdP will not prompt the user for authentication if they already have a valid session. Useful for silent authentication checks.

When checked, the IdP will always require fresh authentication, even if the user has an existing session. Forces re-authentication.

Specify a particular user identifier to test authentication for a specific user. Leave empty to use the IdP's default user selection.
